Пишем parser для https://bmbets.com/
1. Ищу api сайта.
2. В панеле разработчика --> Network просматриваю запросы, ищу какие-либо параметры и заголовки,
которые я могу использовать в своем коде.
3. Ищу в html коде json файлы, которые я смогу использовать.
4. Посмотрев на запросы, увидел, что используется recaptcha. Подумал о том, что в будущем можно сделать телеграм-бота,
который будет выдавать капчу для ввода пользователем.
5. Так как ничего из вышеперечисленного я не нашел, решил парсить html с помощью bs4. Алгоритм придумал такой:
пользователь вводит вид спорта, регион и лигу; в ответ он должен получать строки с матчами и коэффициентами к ним.
a)Получаем данные от пользователя, с интересующим его чемпионатом. Функция get_parse_link возвращает ссылку на страницу
сайта, с запрошенными пользователем данными
б)Изучаем теги, смотрим откуда можно извлечь требуемую информацию. Находим тег и класс, содержащие
нужную нам инфу. Столкнулся с проблемой: в разных дисциплинах(например футбол и снукер) используются разные нужные нам
классы(odds-col3 и odds-col4) - при отсутствии одного из классов ищем по другому классу. Добавляем инфу в list. На выходе
получается 3 списка: 1 - с участниками соревнования, 2 - со всеми
коэффициентами со страницы, в 3 - считываем через join кол-во элементов, равное кол-ву всех коэффициентов/кол-во матчей;
смешиваем список 1 и 3 через zip и выводим на экран.
